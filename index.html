The environment

Simulation you can't fool. Grading you can't game.

01 / PHYSICS

Grounded in real circuit physics

Faults are single resistance or source changes in a full electrical network; every observation is computed, seeded, and deterministic. Load sag, resting monotonicity, and ground-path localization hold by construction — the same invariants that hold under a real multimeter.

02 / GRADING

Cheat-resistant by design

A 0–100 rubric scores root cause, parts discipline, and time efficiency against an expert baseline. Red-herring components bait premature conclusions. Fixes must be verified by a successful start — replacing parts until the symptom disappears earns nothing.

03 / VERIFICATION

Human-verified domain truth

Every scenario is played end-to-end by a domain expert before it ships, with the full fault physics documented and signed off. That review has caught bugs no automated suite flagged — plausible-but-wrong is the failure mode we exist to prevent.

Quickstart

Running in two minutes.

no-start-env is open source and pip-installable. Sanity checks pin every physical invariant; the eval script produces the full scored table for any Inspect-supported model.

$ git clone https://github.com/Jaiparmar940/rlenv
$ pip install -e ".[dev]"
$ python scripts/sanity_check.py # physics invariants
$ python scripts/run_evals.py --scenarios all
# → scored table: root cause · parts · efficiency
